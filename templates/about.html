About Retail Predictions

This web app allows the user to input information about a store department and receive a prediction for weekly sales. The dataset comes from Kaggle and contains anonymized information about 45 retail stores from 2010-2012. The app is built using Flask and the machine learning model is built in Python using the XGBoost learning API.

After inputting values for each variable and clicking "Get Prediction" the input is passed to the model and a prediction is made for that week of sales. In addition, a "force plot" is generated that indicates which variables most influenced the model. Here is an example:


The base value is the mean weekly sales for each store department in the training data on which the model was fit. The average store department does $16027 in sales each week, so the base value is always placed at that spot. The prediction for any store department varies based on the input variables and is where the red and blue meet. The plot shows which variables pushed the prediction to the left or right--that is, which variables caused the model to increase (red) or decrease (blue) its prediction. Note that =1 here means that a categorical variable was selected and =0 means that it was not. This input, therefore, is for Week 47 but not for Department 72.

We see that choosing Week 47 increased the prediction (which makes sense, since it is a holiday week) and not choosing Department 47 (which typically has large sales) decreased the prediction. Try experimenting with different values like Week 51 (right during the holidays) and Department 92 (which has very high sales volume) to see how they impact the prediction.
